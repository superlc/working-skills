# 响应式测试检查清单

> 本清单与 SKILL.md Phase 3「视觉审视清单 V1-V7」配合使用。V1-V7 用于**截图级的快速视觉判定**，本清单用于**逐项深度验证**。每项标注严重程度（🔴 P0 / 🟠 P1 / 🟡 P2）和**检测方式**，帮助快速识别哪些项可自动化检测。

### 检测方式图例

| 图标 | 含义 | 说明 |
|------|------|------|
| 📸 | 截图可检 | 通过分析截图图像即可判定 |
| 📊 | 指标可检 | 通过 layout-metrics.js 采集的量化数据判定 |
| 🔍 | 快照可检 | 通过 accessibility snapshot 的 DOM 结构判定 |
| 👁️ | 人工复查 | 需要人工交互验证（如实际点击、输入、滚动） |
| ❓ | 当前不可自动化 | 受限于工具能力，当前无法自动检测，需人工在真机验证 |

---

## 一、通用检查项（所有端适用）

### 1.1 布局完整性

- [ ] 🔴 📊📸 无水平滚动条溢出（除非设计意图如此，如横向滚动画廊）— `metrics.horizontalOverflow.detected`
- [ ] 🔴 📸 所有核心内容可见，无被裁剪或挤出可视区域的关键信息
- [ ] 🔴 📸 页面无元素大面积重叠、遮挡（小面积装饰性重叠除外）
- [ ] 🟠 📸 页面无异常空白间隙（绝对定位/固定高度在小视口下的常见问题）
- [ ] 🟠 📸 背景图/装饰元素正确适配，无拉伸、错位或空白露出
- [ ] 🟡 📸 页面最小高度铺满视口（短内容页不出现底部大片空白）

### 1.2 文字与排版

- [ ] 🔴 📊📸 文字保持正常横排阅读方向，无被挤压为竖排的情况 — `metrics.typography.suspectedVerticalText`
- [ ] 🔴 📸 文字未溢出容器边界
- [ ] 🟠 📊📸 正文字号在当前视口下可读（mobile ≥ 14px，desktop ≥ 14px，辅助文字 ≥ 12px）— `metrics.typography.minFontSize`
- [ ] 🟠 📸 长文本正确换行，单词/长 URL 不撑破容器（`overflow-wrap: break-word`）
- [ ] 🟠 📸 文字省略处理合理（`text-overflow: ellipsis` 配合 `title` 属性或展开交互）
- [ ] 🟡 📸 行宽合理（正文每行 45–80 字符为佳，避免超宽拉伸降低可读性）
- [ ] 🟡 📸 行高舒适（正文 1.4–1.8 倍行高），段间距清晰
- [ ] 🟡 📸 标题层级视觉分明（h1 > h2 > h3 字号/粗细有递进）

### 1.3 间距与对齐

- [ ] 🟠 📸 内容区域有合理的 padding/margin，不紧贴屏幕边缘（mobile 侧边距 ≥ 16px）
- [ ] 🟠 📸 同级元素间距一致，视觉节奏统一
- [ ] 🟡 📸 元素对齐方式合理（左对齐/居中/右对齐），无错位或参差不齐
- [ ] 🟡 📸 列表项、卡片、网格等重复元素的间距均匀

### 1.4 交互可用性

- [ ] 🔴 📸🔍 核心交互元素（按钮、链接、输入框）可见且可操作
- [ ] 🔴 📸🔍 导航菜单在当前视口下可正常使用（可展开、可点击、可到达所有页面）
- [ ] 🟠 📊 可点击元素尺寸合理（触控设备点击目标 ≥ 44×44px — WCAG 2.5.5）— `metrics.touchTargets`
- [ ] 🟠 📸 相邻可点击元素间距足够（≥ 8px），避免误触
- [ ] 🟠 📸 表单输入框宽度合理，输入内容可完整显示
- [ ] 🟠 📸 弹窗/模态框/抽屉在当前视口下可正常显示和关闭，不超出屏幕
- [ ] 🟠 📸 下拉菜单/选择器展开后不超出视口边界，可完整操作
- [ ] 🟡 👁️ 交互反馈明确（hover/active/focus 状态有视觉变化）
- [ ] 🟡 📸 可滚动区域有明确的视觉提示（渐变遮罩、滚动条、箭头等）

### 1.5 图片与媒体

- [ ] 🟠 📸 图片未变形（保持原始宽高比，使用 `object-fit`）
- [ ] 🟠 📸 图片未溢出容器边界
- [ ] 🟡 📸 图片/图标清晰度与视口 DPR 匹配（Retina 屏使用 2x/3x 资源或 SVG）
- [ ] 🟡 📸 视频/iframe 等嵌入媒体保持宽高比且不溢出

### 1.6 滚动与固定定位

- [ ] 🔴 📸 固定定位元素（fixed header/footer/FAB）不遮挡核心内容，被遮挡内容有足够的 offset
- [ ] 🟠 📸 粘性定位元素（sticky header/sidebar）在滚动时行为正确，不闪烁、不跳动
- [ ] 🟠 📸 页面内嵌可滚动区域（如代码块、长表格）有明确的滚动提示，不与页面滚动冲突
- [ ] 🟡 📸 返回顶部按钮（如有）在长页面中可用且不遮挡内容

### 1.7 加载与状态

- [ ] 🟠 ❓ 页面加载过程中无明显的布局偏移（CLS — Cumulative Layout Shift）— *静态截图无法捕获，需 Lighthouse 或录屏*
- [ ] 🟡 📸 骨架屏/加载指示器的尺寸与最终内容一致，避免加载完成后布局跳动
- [ ] 🟡 📸 空状态（无数据）有合理的占位展示，布局不塌陷
- [ ] 🟡 📸 错误状态（加载失败、网络异常）有友好提示且不破坏布局

### 1.8 无障碍基础

- [ ] 🟠 ❓ 文字与背景色的对比度 ≥ 4.5:1（WCAG AA 标准），大号文字 ≥ 3:1 — *需专用对比度检测工具*
- [ ] 🟠 👁️ 焦点指示器（focus ring）在键盘导航时可见，不被 `outline: none` 移除
- [ ] 🟡 ❓ 页面内容在 200% 缩放下仍可正常使用（无严重布局崩坏）— *需手动缩放测试*
- [ ] 🟡 🔍 图片有 `alt` 文本，装饰性图片使用 `alt=""`

---

## 二、Desktop 专项检查（≥ 1440px）

- [ ] 🟠 📊📸 内容区域宽度有最大限制（`max-width`），避免在超宽屏幕上拉伸到难以阅读 — `metrics.layout.mainArea.widthPercent`
- [ ] 🟠 📸 多列布局正确呈现，列间距适当，内容均匀分布
- [ ] 🟠 📊📸 侧边栏（如有）正常显示，宽度合理，不过度挤压主内容区 — `metrics.layout.navElements`
- [ ] 🟡 👁️ Hover 效果正常工作（下拉菜单、工具提示、按钮反馈）
- [ ] 🟡 📸 表格数据完整展示（无不必要的横向滚动），列宽分配合理
- [ ] 🟡 📸 大屏下留白区域（负空间）分配合理，页面不显空洞或过于密集
- [ ] 🟡 📸 多显示器/超宽屏场景下布局不异常（如 2560px+ 宽度）

---

## 三、Tablet 专项检查（768–1024px）

- [ ] 🔴 📸🔍 导航适配为合适的平板布局（折叠菜单、抽屉式导航或缩略图标式侧边栏）
- [ ] 🟠 📸 多列布局合理调整（如 3→2 列、4→2 列），内容不过度拥挤
- [ ] 🟠 📊📸 侧边栏折叠或转为可收起的抽屉模式，主内容区获得足够宽度 — `metrics.layout.navElements`
- [ ] 🟠 📸 卡片/网格间距合理，不因列数减少而间距异常拉大或压缩
- [ ] 🟠 📸 表格在该宽度下处理合理（横向滚动 / 次要列隐藏 / 卡片化）
- [ ] 🟡 ❓ 横屏（landscape 1024×768）和竖屏（portrait 768×1024）两种方向布局合理 — *需额外横屏配置采集*
- [ ] 🟡 📸 模态框/弹窗宽度适配平板屏幕，不会太窄也不会铺满全屏

---

## 四、Mobile 专项检查（≤ 428px）

> 覆盖 iPhone SE (375px) 到 iPhone 15 Pro Max (428px) 的主流手机视口范围。

### 导航与结构
- [ ] 🔴 📸📊🔍 导航转为汉堡菜单、底部标签栏或全屏导航覆盖层（**侧边栏不能以桌面形态展示**）— `metrics.layout.navElements[].widthPercent > 30%` 为异常
- [ ] 🔴 📸 布局变为单列或少量列（最多 2 列小卡片），无水平布局崩坏
- [ ] 🟠 👁️ 汉堡菜单/抽屉导航的展开和关闭交互正常，有遮罩层且可点击关闭

### 内容与排版
- [ ] 🔴 📸📊 页面核心内容完整可见，不因导航/侧边栏未收起而被挤压到不可用 — `metrics.layout.mainArea.widthPercent`
- [ ] 🟠 📊📸 标题文字不换行为竖排，长标题合理截断或缩小字号 — `metrics.typography.suspectedVerticalText`
- [ ] 🟠 📸 数据统计/标签组等横排元素在窄屏下换行、折叠或支持横向滚动

### 交互适配
- [ ] 🟠 📸 按钮宽度 ≥ 容器宽度或合理堆叠排列（主要操作按钮建议全宽）
- [ ] 🟠 📸 输入框宽度占满可用空间，虚拟键盘弹出后不遮挡当前输入区域
- [ ] 🟠 📸 底部固定操作栏不遮挡页面内容（内容区有足够的 `padding-bottom`）
- [ ] 🟠 📊 触控目标间距足够（相邻可点击元素间距 ≥ 8px）— `metrics.touchTargets`

### 表格与复杂数据
- [ ] 🟠 📸 表格转为卡片式展示、可横向滚动或隐藏次要列
- [ ] 🟡 📸 长列表有虚拟滚动、分页或「加载更多」机制

### 弹窗与浮层
- [ ] 🟠 📸 弹窗/模态框宽度适配屏幕（接近全屏或底部抽屉式）
- [ ] 🟠 📸 弹窗内容可滚动，关闭按钮可触及

### 安全区域
- [ ] 🟠 ❓ 底部操作区域适配 iPhone 安全区域（`env(safe-area-inset-bottom)`），不被 Home Indicator 遮挡 — *Chromium 模拟不含 Home Indicator，需真机验证*
- [ ] 🟡 ❓ 顶部刘海屏/挖孔屏区域不遮挡核心内容 — *Chromium 模拟不含刘海，需真机验证*

## 五、跨端一致性检查

- [ ] 🔴 📸🔍 核心功能在三端均可操作完成（不因适配问题导致功能缺失）
- [ ] 🟠 📸🔍 数据展示完整性一致（不丢失关键信息，次要信息可按端折叠/隐藏）
- [ ] 🟠 👁️ 交互流程在三端可走通（如搜索→筛选→查看详情的链路不中断）
- [ ] 🟡 📸 品牌一致性（颜色、字体、图标风格统一）
- [ ] 🟡 📸 加载态和错误态在三端均有处理
- [ ] 🟡 ❓ 动画/过渡效果在三端表现一致（低性能设备可降级但不应卡顿）— *需录屏或真机验证*

## 六、断点过渡检查

> 断点附近（±50px）的表现往往是响应式实现最脆弱的区域。
> **启用方式**：在 Phase 1 中设置 `breakpoint_test = true`，或在采集脚本中传入中间断点设备配置。

- [ ] 🟠 📸 断点切换时布局过渡自然，无「闪跳」或瞬间布局崩坏
- [ ] 🟠 📸📊 断点临界值处（如 768px、1024px）无元素溢出或异常换行
- [ ] 🟡 📸 在断点之间的中间尺寸（如 500px、600px、900px、1100px、1280px）布局仍合理，无极端拉伸或压缩

**推荐的中间断点测试设备（可通过 `responsive-capture.sh` 的第 4 个参数启用）**：

| 标签 | 宽度 | 测试目标 |
|------|------|---------|
| bp-500 | 500px | 手机横屏 / xs→sm 过渡 |
| bp-600 | 600px | sm→md 过渡 |
| bp-900 | 900px | md→lg 过渡（平板横屏区间） |
| bp-1100 | 1100px | lg→xl 过渡 |
| bp-1280 | 1280px | xl→xxl 过渡 |

## 七、常见响应式问题模式

| 问题 | 症状 | 常见原因 | 严重程度 | 检测方式 |
|------|------|---------|---------|---------|
| 侧边栏未适配 | 侧边栏在移动端仍展开显示，挤压主内容区 | 缺少媒体查询断点隐藏/折叠侧边栏 | 🔴 P0 | 📊📸 |
| 布局崩坏 | 多列内容挤在一起、文字竖排、核心区域不可见 | 未设断点让多列变单列 | 🔴 P0 | 📸 |
| 水平溢出 | 出现横向滚动条 | 固定宽度元素未设 `max-width: 100%`；`width` 使用 px 而非相对单位 | 🔴 P0 | 📊📸 |
| 内容不可见 | 核心内容被挤出屏幕可视区域 | 父容器 `overflow: hidden` + 子元素固定宽度超出 | 🔴 P0 | 📸 |
| 元素重叠 | 两个或多个元素叠在一起无法操作 | `position: absolute/fixed` 未响应式适配；`z-index` 层叠冲突 | 🔴 P0 | 📸 |
| 文字溢出 | 文字超出容器边界 | 缺少 `overflow-wrap: break-word`；容器 `white-space: nowrap` | 🟠 P1 | 📸 |
| 文字竖排 | 标题/文字被挤压为每字一行的竖排 | 容器宽度过小 + 文字不允许换行 | 🟠 P1 | 📊📸 |
| 触控困难 | 按钮/链接难以点击 | 点击区域 < 44px；相邻目标间距 < 8px | 🟠 P1 | 📊 |
| 内容裁剪 | 文字/图片被截断，信息不完整 | `overflow: hidden` + 固定高度，未做自适应处理 | 🟠 P1 | 📸 |
| 固定元素遮挡 | 固定 header/footer 遮挡页面内容 | 内容区缺少对应的 padding/margin offset | 🟠 P1 | 📸 |
| 布局偏移（CLS） | 页面加载中内容突然跳动 | 图片/广告未预留尺寸；字体加载导致回流 | 🟠 P1 | ❓ |
| 空白断层 | 页面中间出现大块空白 | 绝对定位或固定高度在小视口下失效 | 🟡 P2 | 📸 |
| 图片变形 | 图片拉伸或压缩 | 缺少 `object-fit: cover/contain` | 🟡 P2 | 📸 |
| 安全区域遮挡 | iPhone 底部 Home Indicator 遮挡操作按钮 | 缺少 `env(safe-area-inset-bottom)` 适配 | 🟠 P1 | ❓ |
| viewport 单位异常 | 移动端 `100vh` 超出实际可视高度 | 移动浏览器地址栏动态收缩导致 `vh` 不准确，应使用 `dvh` 或 JS 方案 | 🟡 P2 | ❓ |
